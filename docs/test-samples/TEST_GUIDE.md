## 📋 验收清单

### 功能完整性
- [ ] 强匹配用例得分 ≥ 85，recommendation = PROCEED
- [ ] 中等匹配用例得分在 55-75，recommendation = MAYBE
- [ ] 弱匹配用例得分 ≤ 50，recommendation = REJECT
- [ ] dimensions（技能/年限/项目/行业）四项均有分值且合理
- [ ] strengths 至少 2 条，具体指出候选人优势
- [ ] gaps 至少 2 条，具体指出不足之处

### 评分合理性
- [ ] **吴晓峰 × Java后端 = 85-90分**（强匹配，不是60-70）
- [ ] **吴晓峰 × Go后端 = 65-75分**（模糊匹配，Go经验不足）
- [ ] 技术栈偏差：React/Vue差异体现在"技能匹配"维度
- [ ] 能力侧重偏差：数据分析 vs 数据工程的差异被识别
- [ ] 转型候选人：gaps中提到"生产经验不足"而非全盘否定
- [ ] 跨界背景：能识别可迁移技能（如Java→Go的微服务经验）

### AI 洞察深度
- [ ] 能识别简历中的"坑"（如：会写代码 vs 多年未写）
- [ ] 能理解隐含信息（如：独立开发者 → 团队协作经验可能不足）
- [ ] 能平衡理论与实践（如：自学 vs 生产环境使用）
- [ ] 评分理由具体，不是泛泛而谈

## 🚀 快速开始

### 方式 1：API 测试（完整流程）

1. 启动服务

```bash
cd backend && mvn spring-boot:run
```

2. 准备数据（将 txt 手动复制到数据库，或转为 docx/pdf 上传）

3. 测试完整流程

```bash
# 1. 上传并提取简历结构化数据
POST /api/analysis/extract/1

# 2. 批量匹配
POST /api/analysis/match
{
  "jdId": 1,
  "resumeIds": [1, 2, 3, 4, 5]
}

# 3. 查看单个结果
GET /api/analysis/result/1?jdId=1
```

### 方式 2：数据库直接插入（快速验证）

访问 H2 控制台：http://localhost:8080/h2-console

```sql
-- 插入 JD
INSERT INTO job_descriptions (title, content) 
VALUES ('Java后端工程师', '...复制 jd-java-backend.txt 内容...');

-- 插入简历
INSERT INTO resumes (file_name, raw_text) 
VALUES ('张伟_Java高级', '...复制 resume-zhangwei-java-senior.txt 内容...');

-- 获取 ID 后调用 API 进行提取和匹配
```
