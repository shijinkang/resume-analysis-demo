DevOps / SRE 运维开发工程师

岗位职责：
1. 负责公司基础设施建设与运维自动化
2. 搭建和维护 CI/CD 流水线，提升研发效率
3. 管理 Kubernetes 集群，保障线上服务稳定性
4. 设计监控告警体系，快速定位和解决线上问题
5. 进行容量规划和性能优化，降低资源成本
6. 参与系统架构设计，提升系统可靠性和可扩展性
7. 编写运维工具和自动化脚本，减少人工介入

任职要求：
1. 本科及以上学历，计算机相关专业
2. 3年以上运维或 DevOps 工作经验
3. 精通 Linux 系统管理和常用命令，有性能调优经验
4. 熟悉 Docker 和 Kubernetes，有生产环境容器化实践
5. 掌握至少一门脚本语言（Python/Shell/Go）
6. 熟悉 CI/CD 工具（Jenkins/GitLab CI/GitHub Actions/ArgoCD）
7. 了解监控系统（Prometheus/Grafana/ELK/Loki）
8. 熟悉云平台（阿里云/AWS/腾讯云）的基础服务
9. 有 SRE 理念，注重系统可靠性和可观测性

技术栈要求：
- 容器编排：Kubernetes、Docker、Helm
- CI/CD：Jenkins、GitLab CI、ArgoCD、Tekton
- 监控告警：Prometheus、Grafana、Alertmanager、ELK Stack
- 自动化：Ansible、Terraform、Pulumi
- 编程语言：Python/Go/Shell
- 中间件：Nginx、Redis、Kafka、MySQL（运维视角）
- 云平台：熟悉一种主流云厂商的产品

加分项：
- 有大规模集群管理经验（500+ 节点）
- 有混合云或多云架构实践经验
- 熟悉 Service Mesh（Istio/Linkerd）
- 有 Chaos Engineering（混沌工程）实践
- 了解成本优化（FinOps）
- 有开发背景，能看懂业务代码
- CKA/CKS 认证

典型工作内容：
- 优化 CI/CD 流水线，将构建时间从 15 分钟降至 3 分钟
- 搭建 K8s 多集群管理方案，实现灰度发布和蓝绿部署
- 设计分布式链路追踪系统（Jaeger），快速定位微服务问题
- 编写自动扩缩容策略（HPA/VPA/CA），应对流量突增
- 建立 SLA/SLO/SLI 指标体系，保证服务可用性 99.95%

薪资范围：25K - 45K
工作地点：北京/上海/深圳
